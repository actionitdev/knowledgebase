# Perfomance Testing Plan for Solferino WordPress Site

Created by Prithvi (opens new window)

The following documentation explains the recommended strategy to measure the performance of the Solferino academy website which can be configured as one stage in the deployment pipeline. It also covers the performance measurement based on the type of device (PC, phone, etc) used and the location (countries) of the user.

# KPIs to Measure Performance

Following are the Key Performance Indicators (KPI) suggested to measure the performance of general WordPress deployed websites.

  1. Time to interactive
  2. First Visible content
  3. Loading time

# Proposed Technical Solution

It is assumed that CircleCI will be used for the management of the deployment pipeline. Hence, the recommended software/solutions are highly compatible with CircleCI and the docker environment.

Tauras (opens new window) is the recommended solution to write the scripts to measure the performance. Apache Jmeter (opens new window) is one of the most common open-source solutions to load test functional behavior and measure performance. Although it’s quite powerful, it’s not very easy to use and its reporting abilities are very limited. Taurus is a free and open-source framework for Continuous Testing which helps by hiding the complexities of running performance tests.

Following are some references for writing the Tauras scripts which can be used to measure the above-mentioned KPIs for different devices and different locations.

  1. Getting started (opens new window) with Tauras guide.
  2. Tauras-Jmeter (opens new window) covering all the concepts and configuration required to write scenarios (KPI) in Tauras. Refer to this page (opens new window) to refer on how to set the device to simulate the performance (using Blazemeter module).
  3. CircleCI (opens new window) intergration guideline.
  4. Reporting (opens new window) for the interpretation of the results.
  5. Running Tauras through docker (opens new window) scripts.
